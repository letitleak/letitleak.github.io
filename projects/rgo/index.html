rgo: a Go compiler toolchain, written in Rust.

May 21, 2016
compilers rust go

This is a work-in-progress.

My new project, rgo, is a Go compiler toolchain, written in Rust. It’s not finished yet, but it’s basically going to be a program that can compile Go code down to machine code, as well as a library that can be used to parse Go code, among other things.

I started writing rgo because I had a goal: learning how to write a fully-fledged compiler for a programming language that people actuall use. Not a toy language, because, although that would have been much simpler, it would also be a much less rewarding and interesting experience.

I’ve always wanted to write a compiler or an interpreter - something that takes code and turns it into something runnable. Lisp has been reimplemented again and again, and was too simple. C compilers are also commonplace, AND the language has its share of warts.

So I chose to write a compiler for Go. I dislike many of the language’s design choices, but it seemed like a good fit. To the best of my knowledge, there is no reimplementation of the Go compiler, whether in Rust or in another language, so I decided to write one.

There’s an additional challenge: I know very, very little Go, so I’m learning the language along the way. As of writing, I’ve only written a few hundred lines of Go - although I found two bugs in the viper package: #174 and #175.

My main reference material is the Go Specification to look up the syntax of various constructs and translating that into a bunch of Rust structs and enums. This is somewhat tedious, but I’m learning a few things about Go’s syntax along the way.
